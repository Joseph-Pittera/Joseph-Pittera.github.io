Spécialisation: Datas

OLDUVAÏ

Saisons de spécialisation sur les datas : elle dure environ 4 semaines

Nous commençons par voir la validation des données avec Jest, le fameux NTUI (never trust user input). Le but est de vérifier que les données entrées par l'utilisateur sont bien celles attendues afin de ne pas avoir de bugs.

Ensuite nous voyons Sqitch, qui permet de gérer les migrations de base de données. Réinitialiser la base de données, ajouter des colonnes, des tables, des contraintes, etc. mais aussi revenir à une version précédente de la base de données si besoin, de manière simple et rapide.

Puis vient une semaine intensive sur les fonctionnalités avancées du SQL : les DOMAINE, les FONCTIONS, les CHECK via RegExp (expressions régulières), les VUES. SQL commence à ne plus avoir de secrets.

Nous avons quelques jours ensuite sur la documentation JSDoc, et la gestion des erreurs avec Express, Bunyan et Joi, avant de finir cette spécialisation par une semaine d'ouverture sur GraphQL, MongoDB, les Websockets et l'IA.
